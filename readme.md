# Answer Reliability Classification

## Dataset

The dataset contains chatbot conversations, generated answers, retrieved knowledge
chunks, and binary quality labels. Each row represents one answer evaluation case.

| Column | Description |
| --- | --- |
| `full_dialog` | A conversation with the client, split into turns in chronological order. It may contain messages from the client, chatbot, and human operator. |
| `answer` | The chatbot answer to evaluate. |
| `chunk_1`–`chunk_8` | Retrieved relevant knowledge chunks. Some rows contain fewer than eight chunks; unused columns are empty. |
| `binary_relevancy` | Binary answer relevancy label: `1` for a relevant answer and `0` for an irrelevant answer. |
| `binary_faithfulness` | Binary answer faithfulness label: `1` for a truthful answer supported by the knowledge context and `0` otherwise. |
| `markers` | A textual description of an error and its cause. |

### Markers

The `markers` field uses the following error reason labels:

| Marker | Description |
| --- | --- |
| `reason_hallucinated_fact` | The model hallucinated an incorrect fact. |
| `reason_off_topic_answer` | The model answered a different question instead of the user's request. |
| `reason_irrelevant_chunk_used` | The model generated the response using an irrelevant knowledge chunk. |
| `reason_chunk_fact_mixup` | The model incorrectly combined facts from multiple knowledge chunks. |
| `reason_incomplete_answer` | The model provided an incomplete answer and omitted important information. |
| `reason_false_verification` | The model incorrectly claimed to have verified information or performed a calculation. |
| `reason_outdated_fact` | The model used outdated information in its response. |
| `reason_answer_for_operator` | The response contains information intended for the operator rather than the customer. |
| `reason_other` | Other reason. |
| `reason_reveals_ai_identity` | The model revealed or implied that the response was generated by an AI. |
| `reason_wrong_navigation` | The model directed the user to the wrong article, resource, or navigation path. |
| `reason_missed_complaint_handoff` | The model failed to recognize a complaint and did not escalate or hand off the conversation appropriately. |
| `reason_missed_chunk_conditions` | The model failed to use the correct conditions from the relevant knowledge chunk, even though they were available. |

## Task

Given the conversation, the chatbot answer, and the retrieved relevant chunks,
predict whether the answer is reliable.

## Answer Reliability

An answer is **reliable** only when it is both relevant and faithful (truthful):

```text
binary_reliability = binary_relevancy AND binary_faithfulness
```

Therefore, `binary_reliability` is `1` only when both source labels are `1`. In
all other cases it is `0`.

## Binary Relevancy

### Description

Relevancy measures how well the generated answer matches the client's request.

### Evaluation scale

#### Score 1

The answer directly addresses the client's question and helps solve the client's
problem.

#### Score 0

The answer:

- does not address the client's question;
- answers a different but related question;
- contains information intended for the operator;
- contains a clarifying question instead of an answer.

### Evaluation question

Ask: "Is the answer generally on point, and does it provide useful information
that helps solve the client's problem?"

- If yes, assign `1`.
- If no, assign `0`.

### Examples

#### Example 1: relevant but incomplete

**Client request**

> Какие проценты по вкладу «Максимальный» для клиентов А-Клуба?

**Answer**

> На 62 дня — 19.84% с капитализацией.

**Score:** `1`

The answer is incomplete because it gives the rate only for a 62-day deposit,
but it is relevant and directly addresses the request.

#### Example 2: related but not useful

**Client request**

> Какие проценты по вкладу «Максимальный» для клиентов А-Клуба?

**Answer**

> Альфа-Вклад «Максимальный» для клиентов А-Клуба — это вклад с повышенными
> процентными ставками, доступными только для клиентов А-Клуба.

**Score:** `0`

The answer discusses the correct product but does not provide the requested
interest rates.

#### Example 3: redirects instead of answering

**Client request**

> Какие проценты по вкладу «Максимальный» для клиентов А-Клуба?

**Answer**

> Проценты по вкладу можно посмотреть в статье https://abook/article/7621.

**Score:** `0`

The answer redirects the client to an article instead of answering the question.

## Binary Faithfulness

### Description

Faithfulness measures whether the key facts in the generated answer are
consistent with the relevant knowledge context. The comparison focuses on
meaning and key facts rather than exact wording.

### Evaluation scale

#### Score 1

All key facts in the answer—including numbers, conditions, product names, and
other essential information—are supported by the relevant knowledge context.
There is no confusion between products, services, or conditions.

#### Score 0

One or more inconsistencies are present:

- incorrect numbers;
- incorrect conditions or wording that changes the meaning;
- unsupported or incorrect factual information;
- confusion between products or services;
- missing key facts that make the answer misleading or incorrect.

### Evaluation procedure

1. Identify the knowledge chunks relevant to the client's request.
2. Compare the chatbot answer with those chunks.
3. Verify all key facts, numbers, conditions, and product names.
4. Assign `1` only if the answer is fully supported by the relevant context;
   otherwise assign `0`.

### Examples

#### Example 1: incorrect age requirement

**Client request**

> Со скольки лет можно оформить кредитную карту?

**Answer**

> С 14 лет.

**Score:** `0`

The knowledge context states that a credit card is available only to adults
from the age of 18. The answer incorrectly applies a condition associated with
debit cards.

#### Example 2: wrong product

**Client request**

> Сколько стоит кредитная карта?

**Answer**

> Альфа-Карта с гравировкой (Мир): 590 ₽ за первый год обслуживания для
> пакетов "Технический" и "Бесплатные переводы", далее бесплатно.

**Score:** `0`

The answer gives conditions for a debit card rather than the requested credit
card.

#### Example 3: supported answer

**Client request**

> Сколько стоит дебетовая карта?

**Answer**

> Альфа-Карта с гравировкой (Мир): 590 ₽ за первый год обслуживания для
> пакетов "Технический" и "Бесплатные переводы", далее бесплатно.

**Score:** `1`

The numbers, conditions, and product information match the relevant knowledge
context.
